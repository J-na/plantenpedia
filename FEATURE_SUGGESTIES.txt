BESTAANDE SCHEMA-VELDEN DIE NOG NIET ZIJN INGEVULD
--------------------------------------------------------------------
De database heeft al kolommen voor onderstaande velden, maar ze zijn
leeg. Deze toevoegen via data/enrichments/<categorie>.py.
Na toevoegen uploaden met:  python data/upload.py --fields <veldnaam>

  - distribution        → verspreidingsgebied per plant (vrije tekst)
  - fertilizer_needs    → bemesting- en voedingsadvies
  - pruning_info        → snoeiadviezen (wanneer + hoe)
  - weed_behavior       → woekerende of zelfzaaiende eigenschappen
  - pests_diseases      → veelvoorkomende plagen en ziektes + aanpak
  - edible_parts        → welke delen zijn eetbaar
  - recipes             → receptideeën / bereidingswijzen
  - taste               → smaakbeschrijving
  - nutritional_value   → voedingswaarde (bij eetbare planten)
  - toxic_info          → welke delen zijn giftig, voor wie/wat
  - lookalikes          → JSONB: makkelijk te verwarren soorten
  - cultivars           → JSONB: bekende variëteiten met foto
  - sources             → JSONB: wetenschappelijke bronnen
  - wood_properties     → (bomen/struiken) houtkenmerken

  Aanpak: voeg velden toe aan het juiste bestand in data/enrichments/
  voor de meest relevante soorten (bijv. alle eetbare en giftige eerst).
  Upload vervolgens selectief: python data/upload.py --fields edible_parts,toxic_info

1. SEIZOENSKALENDER  (pages/5_Kalender.py)
────────────────────────────────────────────────────────────────────
Idee:
  Visuele weergave van het tuinjaar, met per maand:
  - Welke planten bloeien (kleurgecodeerd per categorie)
  - Onderhoudstaken (snoeien, zaaien, planten, oogsten)

Implementatie-aanwijzingen:
  - Bouw een 12-kolommen tabel (jan–dec) met één rij per plant
  - Gebruik bloom_start / bloom_end uit de database
  - Kleur per categorie (gebruik CATEGORY_LABELS + bijpassende kleur)
  - Filter op categorie via een sidebar-selectbox
  - Optioneel: klik op een cel → spring naar de plantpagina

  Benodigde data die nog ontbreekt:
  - sow_indoors (tuple van maanden, bijv. [2, 3])
  - sow_outdoors (tuple van maanden)
  - plant_out_month (maand van uitplanten)
  - prune_months (lijst van maanden)
  Deze als nieuwe JSONB- of smallint-velden toevoegen aan schema:
    ALTER TABLE plants ADD COLUMN IF NOT EXISTS sow_indoors INT[];
    ALTER TABLE plants ADD COLUMN IF NOT EXISTS sow_outdoors INT[];
    ALTER TABLE plants ADD COLUMN IF NOT EXISTS plant_out_month INT;
    ALTER TABLE plants ADD COLUMN IF NOT EXISTS prune_months INT[];

Prioriteit: HOOG — kernwaarde voor de doelgroep

2. ECOLOGISCHE WAARDE DASHBOARD  (sectie op startpagina of eigen pagina)
────────────────────────────────────────────────────────────────────
Idee:
  Geef elke plant een ecologische-waarde score op basis van bestaande
  data én een nieuw score-veld. Toon top-10 meest ecologisch waardevolle
  planten voor bijen, vlinders, vogels, bodem.

  Score-kolommen toevoegen aan schema:
    ALTER TABLE plants ADD COLUMN IF NOT EXISTS score_insects SMALLINT CHECK (score_insects BETWEEN 0 AND 5);
    ALTER TABLE plants ADD COLUMN IF NOT EXISTS score_birds   SMALLINT CHECK (score_birds   BETWEEN 0 AND 5);
    ALTER TABLE plants ADD COLUMN IF NOT EXISTS score_soil    SMALLINT CHECK (score_soil    BETWEEN 0 AND 5);

  De totaalscore = som van de drie. Automatisch berekend in Python.

  Op de startpagina: een "Top 5 voor bijen deze maand" sectie die
  filtert op bloeiende planten (huidige maand) met hoogste score_insects.

Implementatie-aanwijzingen:
  - Voeg score-velden toe aan data/enrichments/<categorie>.py per plant
  - Upload: python data/upload.py --fields score_insects,score_birds,score_soil
  - Maak een component display_eco_score(plant) in utils/display.py
    die ★★★☆☆ sterren toont of kleurbalken gebruikt
  - Op de plantdetailpagina tonen naast de bestaande badges

Prioriteit: MIDDEL

DEEL 3 — DATA-UITBREIDINGEN IN ENRICHMENTS-BESTANDEN
====================================================================

Velden toevoegen aan data/enrichments/<categorie>.py en uploaden met
python data/upload.py --fields <veldnaam>  (of --safe voor alle).

NIEUWE VELDEN (vereisen eerst ALTER TABLE in Supabase SQL Editor):

  native_nl         : True/False — inheems in Nederland/België?
    SQL: ALTER TABLE plants ADD COLUMN IF NOT EXISTS native_nl BOOLEAN DEFAULT FALSE;

  drought_tolerant  : True/False — droogtebestendig?
    SQL: ALTER TABLE plants ADD COLUMN IF NOT EXISTS drought_tolerant BOOLEAN DEFAULT FALSE;

  water_needs       : 'droog' | 'normaal' | 'vochtig' | 'nat'
    SQL: ALTER TABLE plants ADD COLUMN IF NOT EXISTS water_needs TEXT
         CHECK (water_needs IN ('droog', 'normaal', 'vochtig', 'nat'));

  spacing_cm        : plantafstand in cm (integer)
    SQL: ALTER TABLE plants ADD COLUMN IF NOT EXISTS spacing_cm SMALLINT;

  sow_indoors       : lijst van maandnummers, bijv. [2, 3]
    SQL: ALTER TABLE plants ADD COLUMN IF NOT EXISTS sow_indoors INT[];

  sow_outdoors      : lijst van maandnummers, bijv. [4, 5]
    SQL: ALTER TABLE plants ADD COLUMN IF NOT EXISTS sow_outdoors INT[];

  plant_out_month   : maandnummer, bijv. 5
    SQL: ALTER TABLE plants ADD COLUMN IF NOT EXISTS plant_out_month SMALLINT;

  prune_months      : lijst van maandnummers, bijv. [3, 8]
    SQL: ALTER TABLE plants ADD COLUMN IF NOT EXISTS prune_months INT[];

  score_insects     : 0–5 (0 = geen waarde, 5 = uitstekend voor insecten)
  score_birds       : 0–5
  score_soil        : 0–5
    SQL: ALTER TABLE plants ADD COLUMN IF NOT EXISTS score_insects SMALLINT CHECK (score_insects BETWEEN 0 AND 5);
         ALTER TABLE plants ADD COLUMN IF NOT EXISTS score_birds   SMALLINT CHECK (score_birds   BETWEEN 0 AND 5);
         ALTER TABLE plants ADD COLUMN IF NOT EXISTS score_soil    SMALLINT CHECK (score_soil    BETWEEN 0 AND 5);

BESTAANDE VELDEN (bestaan al in DB, geen SQL nodig — gewoon toevoegen aan enrichments):

  distribution      : verspreidingsgebied beschrijving
  fertilizer_needs  : bemestingsadvies
  pruning_info      : snoeiadvies
  weed_behavior     : agressieve spreiding / zelfzaai
  pests_diseases    : veelv. plagen + ecologische aanpak
  edible_parts      : welke delen eetbaar (voor eetbare soorten)
  recipes           : 1–2 bereidingsideeën
  toxic_info        : welke delen giftig + symptomen (voor giftige soorten)
  cultivars         : bekende cultivars (JSONB)

DEEL 4 — TECHNISCHE VERBETERINGEN
====================================================================

  1. Caching verfijnen
     - Huidige @st.cache_data TTL = 3600 sec
     - Overweeg TTL = 86400 (1 dag) voor plantdata die zelden wijzigt
     - Admin-wijzigingen moeten cache invalideren (al geïmplementeerd
       via cache.clear() — controleren of dit goed werkt)

  2. Full-text search verbeteren
     - Huidige search_plants() zoekt via ilike op scientific_name +
       dutch_names. Voeg ook description en growth_habit toe aan de
       zoek-query voor betere resultaten.

  3. Sitemap / SEO
     - Genereer een sitemap.xml zodat zoekmachines alle plantpagina's
       kunnen indexeren (relevant als de app publiek wordt)

  4. Responsive layout
     - Testen op mobiel (Streamlit is mobiel-bruikbaar maar niet
       geoptimaliseerd). Overweeg CARDS_PER_ROW dynamisch maken:
         cols = 1 if screen_width < 600 else (2 if < 900 else 4)
       (vereist een JS-hack of Streamlit component)

  5. Foto fallback
     - In utils/display.py: wrap st.image() in try/except al gedaan;
       voeg een placeholder-SVG toe als fallback
